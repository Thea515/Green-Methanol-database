# 快速部署指南

## 第一步:创建GitHub仓库 (5分钟)

### 1.1 访问GitHub
打开浏览器,访问: https://github.com/new

### 1.2 创建仓库
填写以下信息:
- Repository name: `green-methanol-database`
- Description: `全球绿色甲醇项目数据库 - 实时更新的项目信息平台`
- 选择 **Public** (公开仓库)
- 勾选 **Add a README file**
- 点击 **Create repository** 按钮

## 第二步:上传文件 (5分钟)

### 2.1 方式一:通过网页上传(最简单)

1. 在新创建的仓库页面,点击 **Add file** > **Upload files**
2. 将以下文件从本地拖拽到上传区域:
   - `index.html`
   - `styles.css`
   - `script.js`
   - `data.json`
   - `convert.js`
   - `README.md`
3. 添加提交信息: `Initial commit`
4. 点击 **Commit changes**

### 2.2 方式二:通过Git命令上传(推荐)

```bash
# 进入项目目录
cd /Users/tivyc/WorkBuddy/20260312112501/web

# 初始化Git仓库
git init

# 添加远程仓库(替换YOUR_USERNAME为你的GitHub用户名)
git remote add origin https://github.com/YOUR_USERNAME/green-methanol-database.git

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit"

# 推送到GitHub
git branch -M main
git push -u origin main
```

## 第三步:启用GitHub Pages (2分钟)

### 3.1 进入Pages设置
1. 在仓库页面,点击 **Settings** 标签
2. 在左侧菜单中找到并点击 **Pages**

### 3.2 配置Pages
1. 在 **Build and deployment** 部分
2. **Source** 选择: **GitHub Actions**
3. 页面会自动保存

## 第四步:创建GitHub Actions配置 (3分钟)

### 4.1 创建工作流文件
在本地创建目录结构:

```bash
mkdir -p .github/workflows
```

### 4.2 创建update.yml文件
将以下内容保存到 `.github/workflows/update.yml`:

```yaml
name: Update Database and Deploy

on:
  schedule:
    - cron: '0 2 * * 1'  # 每周一凌晨2点(UTC)运行
  workflow_dispatch:     # 支持手动触发

jobs:
  update:
    runs-on: ubuntu-latest
    
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '18'
      
      - name: Run update script
        run: |
          echo "开始更新数据库..."
          node convert.js
      
      - name: Commit and push changes
        run: |
          git config --local user.email "github-actions[bot]@users.noreply.github.com"
          git config --local user.name "github-actions[bot]"
          git add data.json
          git diff --quiet && git diff --staged --quiet || git commit -m "Auto update database [$(date +'%Y-%m-%d')]"
          git push
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

### 4.3 提交配置文件

```bash
git add .github/workflows/update.yml
git commit -m "Add GitHub Actions workflow"
git push
```

## 第五步:访问网页 (立即)

### 5.1 等待部署
1. 在仓库页面,点击 **Actions** 标签
2. 等待工作流完成(约2-3分钟)
3. 看到 ✅ 绿色勾号表示成功

### 5.2 访问网页
访问地址: `https://YOUR_USERNAME.github.io/green-methanol-database/`

(将 YOUR_USERNAME 替换为你的GitHub用户名)

## 第六步:配置自动更新 (可选)

### 6.1 手动触发更新
1. 进入 **Actions** 标签
2. 选择 **Update Database and Deploy** 工作流
3. 点击右侧 **Run workflow** 按钮
4. 点击 **Run workflow** 确认

### 6.2 修改更新频率
编辑 `.github/workflows/update.yml` 中的 cron 表达式:

```yaml
on:
  schedule:
    - cron: '0 2 * * 1'  # 每周一凌晨2点
    # - cron: '0 2 * * *'  # 每天凌晨2点
    # - cron: '0 2 1 * *'  # 每月1日凌晨2点
```

## 常见问题

### Q1: 网页显示404错误
**A**: 等待2-3分钟,GitHub Pages需要时间部署。如果还是404,检查:
1. 仓库是否设置为Public
2. GitHub Actions是否成功运行
3. 文件路径是否正确

### Q2: 图表不显示
**A**: 检查:
1. `data.json` 文件是否存在且格式正确
2. 浏览器控制台是否有错误(F12打开开发者工具)
3. 网络连接是否正常(需要访问CDN加载Chart.js)

### Q3: 地图不显示
**A**: 确保项目数据中的`location`字段包含有效的地理位置,并在`script.js`的`coordinates`对象中添加对应坐标。

### Q4: GitHub Actions运行失败
**A**: 查看:
1. Actions标签中的运行日志
2. 检查`convert.js`文件是否存在
3. 确保Node.js版本为18或更高

### Q5: 如何更新数据
**A**: 有三种方式:
1. **自动更新**: 等待每周一的定时任务
2. **手动触发**: 在Actions页面点击"Run workflow"
3. **本地更新**: 编辑CSV后运行`node convert.js`,然后git push

## 下一步优化

### 添加自定义域名
1. 在GitHub仓库的Settings > Pages中
2. 添加自定义域名(如 `methanol.yourdomain.com`)
3. 配置DNS记录:
   - A记录: 指向 `185.199.108.153`
   - CNAME: 指向 `YOUR_USERNAME.github.io`

### 添加Google Analytics
1. 注册Google Analytics账号
2. 获取跟踪代码
3. 在`index.html`的`</head>`前添加:
```html
<script async src="https://www.googletagmanager.com/gtag/js?id=GA_TRACKING_ID"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'GA_TRACKING_ID');
</script>
```

### 添加评论系统
1. 使用Disqus或Utterances
2. 在网页中嵌入评论代码
3. 允许访客留言讨论
